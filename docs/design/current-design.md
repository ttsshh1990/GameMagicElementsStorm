# Current Design

最后更新：2026-04-24

## 当前一句话概念

以冰 / 火 / 雷三元素收集、合成与强化为核心 build 乐趣的弹幕 Roguelike。

## 目标体验

- 玩家在局内持续围绕“拿什么元素、要不要立刻合成、追哪条高级元素路线、为哪种符文调整路径”做取舍。
- 同一套三元素配方可以因为元素强化和符文激活差异，长出不同战斗表现和 build 变体。
- 每个技能都应具备发展成主要核心技能的潜力，组合差异主要来自技能形态、视觉效果和造成伤害方式，而不是预设谁是主输出或辅助路线。

## 核心玩法假设

- 基础玩法是 2D 俯视角弹幕射击，玩家主要负责移动走位，技能以内置 CD 自动释放。
- 战斗场地先采用一个稍大的平面房间，不做复杂地形、房间切换或波次结算。
- 怪物无限出现并随时间和玩家等级逐渐变强，玩家通过击杀怪物、拾取经验晶体升级。
- 升级时暂停战斗并弹出奖励选择，玩家在奖励界面中做 build 决策。
- 开局直接拥有 3 个基础元素：冰、火、雷。
- 升级奖励主要提供三类成长：获得新元素、强化元素、获得符文类升级。
- 任意 3 个元素组成一个配方，可以合成为 1 个高级元素技能；第一次合成会消耗原来的 3 个基础元素。
- 元素合成在玩家达到 3 级后解锁。
- 同一路线的高级元素可以继续进行二次和三次三连合一，分别成长为紫色品质和橙色品质版本。
- 元素强化不只影响基础元素本身，还会影响后续由该元素参与合成出的高级技能表现。
- 符文是局内 build 深化系统，获得后不会自动生效，需要满足元素激活条件。
- 数值使用全局基准值和倍率成长，避免为每个技能、每个怪物、每一级单独 hardcode 数值。

## 玩家循环

1. 开局带着冰、火、雷 3 个基础元素进入战斗。
2. 在单个大平面房间中移动走位，自动释放当前技能，躲避持续出现并逐渐变强的怪物。
3. 击杀怪物后掉落经验晶体，玩家通过走位拾取经验晶体填充经验条。
4. 升级时暂停战斗，弹出奖励选择，从新元素、元素强化、符文等奖励中选择。
5. 3 级后解锁元素合成，玩家开始围绕当前元素槽、未来配方目标和符文激活条件调整 build。
6. 通过二次、三次合成把核心技能推进到紫色和橙色阶段，在无限压力中尽量延长生存时间。

## 关键系统

### 战斗结构与操作

- 视角：2D 俯视角。
- 场地：一个稍大的平面房间，先不设计多房间、障碍物和复杂地形。
- 玩家操作：移动走位为主，技能不需要玩家手动瞄准或释放。
- 技能释放：每个技能有内置 CD，到时间后自动释放。
- 目标选择：原型阶段优先使用最近敌人、玩家周围随机点、玩家当前位置或技能自身运动逻辑。
- 敌人结构：怪物无限出现，随时间和玩家等级逐渐提高数量与强度。
- 奖励节奏：取消波次结算，改为拾取经验晶体升级；升级时暂停战斗并弹出奖励选择。

### 基础元素与初始状态

- 基础元素为冰、火、雷。
- 开局直接拥有 3 个基础元素：冰、火、雷。
- 玩家同时只装备 3 个元素槽，元素槽可放任意级别元素：基础 / 高级 / 紫色 / 橙色。

### 三元素合成

- 第一次合成：3 个元素组成一个配方，产出 1 个高级元素技能。
- 第一次合成后，参与配方的 3 个基础元素会被消耗。
- 玩家达到 3 级后才解锁元素合成。
- 当前按 10 种三元素组合来设计对应技能或技能形态。

### 经验、升级与奖励

- 怪物死亡后掉落经验晶体。
- 玩家需要移动到经验晶体附近拾取，拾取后增加经验值。
- 经验值达到升级阈值后，战斗暂停并弹出奖励选择。
- 每次升级提供 3 选 1 奖励。
- 1-2 级奖励池不提供元素合成操作，主要用于获得元素、强化元素或基础成长。
- 3 级后解锁元素合成，并允许玩家在升级奖励界面中处理合成相关决策。

### 品质成长阶段

- 第一次三元素合成：获得 1 个高级元素技能，完成一次技能成型。
- 第二次合成：3 个相同高级元素合成 1 个紫色品质高级元素，提供一次明显强度飞跃。
- 第三次合成：3 个相同紫色高级元素合成 1 个橙色品质高级元素，提供终极质变进化。

### 元素强化

- 升级奖励可直接强化某个基础元素等级。
- 元素等级越高，与该元素相关的高级技能越强。
- 当前强化方向共识：
  - 冰：更强控场、减速、冻结与防线稳定性。
  - 火：更高爆发、更大范围、更快清杂与直接伤害结算。
  - 雷：更多连锁、更高触发频率、更快传播与更强群体清理效率。

### 符文系统（暂定名）

- 符文是升级奖励中的一类升级，当前设定可有不同品质。
- 低品质偏基础提升，如伤害、暴击、范围；高品质偏质变效果，如直接改写技能行为或触发方式。
- 当前版本不设装备上限，玩家获得的所有符文只要满足激活条件，都可以同时生效。
- 符文激活条件只按当前装备的 3 个元素槽计算，不看历史配方和背包库存。
- 当前拆算规则：
  - 基础元素 = 1
  - 高级元素 = 3
  - 紫色高级元素 = 9
  - 橙色高级元素 = 27
- 当前符文激活总数建议：
  - 普通：3~4
  - 精良：5~7
  - 稀有：8~11
  - 史诗：12~16
  - 传说：18~24
  - 终局：27+
- 当前设计原则是“品质先决定总激活数，再在总数内随机分配火 / 冰 / 雷占比”；低品质条件偏宽松，高品质条件可更偏科。

### 数值规则

数值系统使用少量全局基准值和倍率公式。技能伤害、怪物血量、怪物强度、升级经验都围绕基准值计算，后续调参优先调整全局参数和倍率，而不是逐个技能或逐级写死数值。

#### 核心基准值

| 参数 | 默认值 | 含义 |
|---|---:|---|
| `BASE_DPS` | 10 | 一个标准技能在基础状态下的期望每秒输出 |
| `TARGET_TTK` | 3.0 | 普通怪在基准输出下的目标死亡时间，单位秒 |
| `ENEMY_HP_BASE` | 30 | 普通怪基础血量，等于 `BASE_DPS * TARGET_TTK` |
| `TIME_GROWTH` | 0.10 | 每分钟带来的怪物强度成长 |
| `LEVEL_GROWTH` | 0.04 | 玩家每升 1 级带来的怪物强度成长 |
| `ELEMENT_LEVEL_BONUS` | 0.12 | 参与配方的元素每升 1 级，对相关技能提供的伤害成长 |
| `CONTACT_DAMAGE_BASE` | 10 | 普通怪基础接触伤害 |
| `SPAWN_RATE_BASE` | 1.0 | 基础刷怪频率，每秒 1 只 |
| `MAX_ALIVE_BASE` | 20 | 基础场上最大怪物数 |
| `MAX_ALIVE_CAP` | 90 | 原型阶段场上最大怪物数上限 |
| `XP_BASE` | 20 | 从 1 级升到 2 级需要的经验 |
| `XP_GROWTH` | 1.22 | 每级经验需求成长倍率 |
| `XP_DROP_BASE` | 1 | 普通怪基础经验掉落 |

#### 怪物强度公式

```text
T = 当前游戏时间，单位分钟
L = 玩家等级

threat = 1 + TIME_GROWTH * T + LEVEL_GROWTH * (L - 1)
```

普通怪血量：

```text
enemy_hp = ENEMY_HP_BASE * enemy_role_hp_multiplier * threat
```

接触伤害：

```text
enemy_contact_damage = CONTACT_DAMAGE_BASE * enemy_role_damage_multiplier * (1 + 0.06 * T)
```

刷怪频率：

```text
spawn_rate = SPAWN_RATE_BASE * (1 + 0.12 * T + 0.03 * (L - 1))
```

场上最大怪物数：

```text
max_alive = min(MAX_ALIVE_CAP, MAX_ALIVE_BASE + floor(2.5 * T) + floor(1.5 * (L - 1)))
```

#### 怪物类型倍率

| 怪物类型 | HP 倍率 | 速度倍率 | 伤害倍率 | XP 倍率 |
|---|---:|---:|---:|---:|
| 普通追踪怪 | 1.0 | 1.0 | 1.0 | 1.0 |
| 小型快怪 | 0.45 | 1.35 | 0.7 | 0.6 |
| 厚血怪 | 2.5 | 0.7 | 1.4 | 2.0 |

#### 技能伤害公式

每个技能只定义 CD、伤害 profile、期望命中次数、品质、参与配方元素等级等少量参数。每次释放的总伤害预算：

```text
build_power = quality_multiplier * element_power * rune_power
cast_damage_budget = BASE_DPS * cooldown * build_power * profile_multiplier
```

单次命中伤害：

```text
hit_damage = cast_damage_budget / expected_hit_count
```

持续伤害 tick：

```text
tick_damage = cast_damage_budget / expected_tick_count
```

元素强化：

```text
element_power = 1 + ELEMENT_LEVEL_BONUS * average(recipe_element_level - 1)
```

品质倍率：

| 品质 | 倍率 |
|---|---:|
| 基础 / 高级 | 1.0 |
| 紫色 | 1.75 |
| 橙色 | 3.0 |

技能伤害 profile：

| Profile | 适用技能 | 期望命中 / tick | Profile 倍率 |
|---|---|---:|---:|
| `single_lock` | 镭射、单体锁定 | 1.5 | 1.0 |
| `area_burst` | 陨石、冰锥区域 | 3 | 1.1 |
| `chain` | 闪电链 | 4 | 1.0 |
| `moving_orb` | 电球 | 5 | 1.15 |
| `dot_field` | 熔岩、持续区域 | 8 | 1.2 |
| `aura` | 环绕能量场 | 按 tick 计算 | 0.35 DPS / 目标 |

#### 经验公式

升级所需经验：

```text
xp_to_next_level = XP_BASE * XP_GROWTH ^ (L - 1)
```

怪物经验掉落：

```text
xp_drop = XP_DROP_BASE * enemy_role_xp_multiplier
```

## 当前不确定点

- 冰冰火、火火冰这 2 个组合仍需重做。
- 冰冰雷、火火雷、雷雷雷、雷雷冰、雷雷火、冰火雷的技能细节仍需继续细化。
- 1-2 级的基础技能表现仍需明确，尤其是未合成前冰、火、雷三个基础元素如何自动攻击。
- 3 级后元素合成在 UI 中的具体操作方式仍需定义。
- 第二次紫色品质阶段，除了数值飞跃外，是否附带少量形态变化仍待确认。
- 第三次橙色品质阶段的“质变进化”定义仍待细化。
- 符文系统的最终命名仍未确定。
- 同类符文使用不同激活条件时，强度是否需要做补偿仍待确认。
- 升级奖励中“新元素”“元素等级强化”“符文奖励”的比例尚未设计完成。
- 当玩家拿到强力但难激活的符文时，系统应该提供多少追条件机会仍待定义。
- 目标平台、美术方向和世界观主题仍未展开。

## 暂缓方向

- 暂不创建 Godot 工程。
- 暂不实现玩法原型。
- 暂不生成正式美术资源。
- 暂不扩展额外设计文档体系，继续只维护 `current-design.md` 与 `memory.md`。
